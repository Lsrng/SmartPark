# 智慧停车系统 — 前端开发指引文档

> 本文档用于指导 AI 生成基于 Vue 3 + JavaScript + Element Plus 的前端页面，调用后端 smartPark 项目的 RESTful API。

---

## 二、后端接口文档

### 2.1 通用说明

**基础地址**：`http://localhost:8086`

**请求头要求**：
- 登录接口：`Content-Type: application/json`
- 其他接口：`Content-Type: application/json` + 需要在请求头中携带 Token

**统一响应格式**：

```json
{
    "code": 200,
    "msg": "操作成功",
    "data": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，200 成功，401 未授权，429 限流，500 服务端错误 |
| msg | String | 提示信息 |
| data | T | 响应数据，可能为 null |

---

### 2.2 用户认证

#### POST /auth/login — 用户登录

**请求体**：

```json
{
    "username": "admin",
    "password": "123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**成功响应**（200）：

```json
{
    "code": 200,
    "msg": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiJ9...",
        "tokenName": "token",
        "user": {
            "id": 1,
            "username": "admin"
        }
    }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| data.token | String | JWT 令牌，后续接口需要放在请求头中 |
| data.tokenName | String | Token 的请求头名称，值为 `token` |
| data.user | Object | 用户基本信息 |

**失败响应**（401）：

```json
{
    "code": 401,
    "msg": "用户名或密码错误",
    "data": null
}
```

> **前端操作指引**：登录成功后，将 `token` 存入 localStorage 或 Pinia，后续所有请求在请求头中携带 `token: <value>`。

### 2.3 物联设备数据上报

#### POST /device/data/report — 上报设备数据

**请求头**：需要携带 JWT Token

```http
token: eyJhbGciOiJIUzI1NiJ9...
Content-Type: application/json
```

**请求体**：

```json
{
    "deviceId": 1,
    "data": {
        "temperature": 25.6,
        "humidity": 60,
        "status": "normal"
    },
    "reportTime": "2026-07-20T10:30:00"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| deviceId | Long | 是 | 设备 ID |
| data | Object | 是 | 上报数据内容（任意 JSON 对象） |
| reportTime | String (datetime) | 否 | 设备上报时间，不传则使用服务器时间，格式：yyyy-MM-ddTHH:mm:ss |

**成功响应**（200）：

```json
{
    "code": 200,
    "msg": "上报成功",
    "data": null
}
```

**限流响应**（429）：

```json
{
    "code": 429,
    "msg": "设备 1 上报过于频繁，请稍后重试（限制：每分钟 1 次）",
    "data": null
}
```

**未授权响应**（401）：

```json
{
    "code": 401,
    "message": "Token无效或已过期"
}
```

## 三、前端页面需求

### 页面 1：登录页面

**路由**：`/login`

**功能说明**：

| 功能点 | 说明 |
|--------|------|
| 用户名输入框 | 文本输入，不能为空 |
| 密码输入框 | 密码类型输入，不能为空 |
| 登录按钮 | 点击后调用 `/auth/login` 接口 |
| 登录成功 | 将 token 存入 localStorage，跳转到首页 |
| 登录失败 | 显示后端返回的错误信息 |

**页面样式**：居中卡片式登录表单，带系统名称标题。

---

### 页面 2：设备数据上报页面

**路由**：`/device/report`

**功能说明**：

| 功能点 | 说明 |
|--------|------|
| 设备 ID 输入框 | 数字输入，必填 |
| 上报数据编辑器 | 文本域或 JSON 编辑器，输入 JSON 格式数据，必填 |
| 上报时间选择器 | 日期时间选择器，选填，默认当前时间 |
| 提交按钮 | 点击后调用 `/device/data/report` 接口 |
| 提交成功 | 显示成功提示，清空表单或保留表单 |
| 被限流 | 显示 429 错误提示（限流信息） |
| token 过期 | 跳转到登录页 |

**页面样式**：表单布局，带提交按钮，响应式。

## 四、前端核心代码要求

### 4.1 Axios 请求封装

需要创建一个统一的 Axios 实例：

```javascript
// src/utils/request.js

import axios from 'axios';
import { ElMessage } from 'element-plus';
import router from '@/router';

const request = axios.create({
    baseURL: 'http://localhost:8086',
    timeout: 10000
});

// 请求拦截器：自动携带 token
request.interceptors.request.use(
    config => {
        const token = localStorage.getItem('token');
        if (token) {
            config.headers['token'] = token;
        }
        return config;
    },
    error => Promise.reject(error)
);

// 响应拦截器：统一处理错误
request.interceptors.response.use(
    response => {
        const res = response.data;
        // 业务状态码处理
        if (res.code === 401) {
            localStorage.removeItem('token');
            router.push('/login');
            ElMessage.error(res.msg || '登录已过期');
            return Promise.reject(new Error(res.msg));
        }
        if (res.code !== 200) {
            ElMessage.error(res.msg);
            return Promise.reject(new Error(res.msg));
        }
        return res;
    },
    error => {
        // HTTP 状态码处理
        if (error.response) {
            const status = error.response.status;
            if (status === 429) {
                ElMessage.warning(error.response.data?.msg || '请求频率过高');
            } else if (status === 401) {
                localStorage.removeItem('token');
                router.push('/login');
            } else {
                ElMessage.error('服务器错误');
            }
        } else {
            ElMessage.error('网络错误');
        }
        return Promise.reject(error);
    }
);

export default request;
```

### 4.2 路由守卫

需要实现路由守卫，未登录时跳转到登录页：

```javascript
// src/router/index.js

import { createRouter, createWebHistory } from 'vue-router';

const routes = [
    {
        path: '/login',
        name: 'Login',
        component: () => import('@/views/Login.vue'),
        meta: { requiresAuth: false }
    },
    {
        path: '/',
        name: 'Home',
        component: () => import('@/views/Home.vue'),
        meta: { requiresAuth: true }
    },
    {
        path: '/device/report',
        name: 'DeviceReport',
        component: () => import('@/views/DeviceReport.vue'),
        meta: { requiresAuth: true }
    }
];

const router = createRouter({
    history: createWebHistory(),
    routes
});

// 路由守卫
router.beforeEach((to, from, next) => {
    const token = localStorage.getItem('token');
    if (to.meta.requiresAuth && !token) {
        next('/login');
    } else {
        next();
    }
});

export default router;
```

## 六、关键交互流程

```
用户访问页面
    │
    ├── 未登录 → 显示登录页
    │               │
    │               ├── 输入用户名密码 → POST /auth/login
    │               │       ├── 成功 → 存 token，跳转首页
    │               │       └── 失败 → 显示错误提示
    │               │
    └── 已登录 → 显示首页（控制台）
                    │
                    ├── 导航到设备数据上报
                    │       ├── 填写设备 ID、数据内容等
                    │       ├── POST /device/data/report
                    │       │       ├── 200 → 上报成功提示
                    │       │       ├── 429 → 限流提示
                    │       │       └── 401 → token 过期 → 跳转登录页
                    │
                    ├── 退出登录 → 清除 token，跳转登录页
```

## 七、注意事项

### 7.1 Token 管理
- Token 名称在后端配置为 `token`，请求头名称为 `token`
- 登录响应中的 `tokenName` 字段值为 `token`，可直接用于请求头
- Token 过期时间为 4 小时

### 7.2 限流体验
- 设备上报接口每个设备每分钟限流 1 次
- 前端无需做额外的限流逻辑，后端返回 429 时直接展示提示即可

### 7.3 跨域问题
- 后端已配置 CORS（[CorsConfig.java](file:///e:/后端/项目练习/smartPark/src/main/java/com/smartpark/config/CorsConfig.java)），支持跨域访问
- 前端开发时 Vite 代理不是必需的，但也可以配置 proxy 避免跨域

### 7.4 设备 ID 类型
- `deviceId` 为 Long 数字类型，不是字符串

## 八、Vite 代理配置（可选）

如果希望避免跨域，可以在 `vite.config.js` 中配置代理：

```javascript
export default defineConfig({
    plugins: [vue()],
    server: {
        port: 5173,
        proxy: {
            '/auth': {
                target: 'http://localhost:8086',
                changeOrigin: true
            },
            '/device': {
                target: 'http://localhost:8086',
                changeOrigin: true
            }
        }
    }
});
```

使用代理后，Axios 的 `baseURL` 可改为 `''` 或 `'/'`。
